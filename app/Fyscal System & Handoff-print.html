01 · Final architecture

One app, layered cleanly

A single React app rendered into a fixed iPhone frame. No build step — Babel transpiles in-browser so the whole thing is portable. Every module is a screen factory registered in one router; all share the same tokens, primitives, icons, and mock-data layer.

Foundation

tokens.css · app.css — the Fyscal palette, type scale, spacing, radii, shadows and every component class.

Primitives

ui.jsx — StatusBar, AppBar, BottomNav, Sheet, Dialog, Toast, Keypad, PinDots, Avatar, StepBar, Switch, money + sound helpers.

Icons

icons.jsx — one stroke-based set (~70 glyphs), currentColor, sized by prop. No emoji in UI chrome.

Data

data.jsx — a single window.DATA store: user, banks, payees, transactions, billers, rewards, tickets, notifications.

Shell + router

app.jsx — stack navigation, tab switching, status-bar tone, toast host, persistence.

Modules

11 screens_*.jsx files, each exporting a window.*Screens object of screen components.

02 · Navigation system

One stack, five tabs

A single navigation stack drives the whole app. Five persistent tabs form the spine; every other screen pushes on top and pops back. State persists to localStorage so a refresh restores the session.

The nav contract

nav.go(screen, params)Push a screen (tabs reset the stack)
nav.back()Pop one level
nav.replace(...)Swap the top (e.g. verify → amount)
nav.reset(screen)Clear stack to one root (post-payment → home)

Tab spine

🏠 Home🕑 History⧉ Scan (FAB)🧾 Bills👤 Profile

Center Scan is a raised FAB — the camera-first action. Switching tabs resets that tab's stack so users never get lost deep in a flow.

Global behaviors wired into the shell

  • ✓Status-bar tone auto-flips dark/light per screen (light over camera, splash, success gradients).
  • ✓Toast host + auth state live at the shell and are passed to every screen as app.
  • ✓Cold start → splash, which routes to Home (returning) or Onboarding (new) based on auth.
03 · Design consistency audit

Everything resolves to tokens

No module hardcodes a stray colour, font, or radius. Spacing, type, cards, forms, buttons, sheets, modals and nav all draw from one scalable system.

Standardised & consistent

  • ✓Type — Plus Jakarta Sans display + JetBrains Mono for all figures & IDs.
  • ✓Radii — 14px inputs, 16–18px cards/rows, 22px hero blocks, pill for chips.
  • ✓Cards & rows — one .card/.list-card/.row vocabulary everywhere.
  • ✓Tinted icon tiles — color-mix of a semantic colour at 12% on white, used in every list.
  • ✓Money — single inr()/inrParts() formatter (Indian grouping).

Conventions to hold the line on

  • !Destructive actions always go through a Dialog confirm (logout, remove bank/device, deactivate UPI, close account).
  • !PIN entry always uses PinDots + Keypad in a bottom Sheet.
  • !Every result state reuses the check-ring draw + (success) confetti + chime.

06 · Global UX behaviors

Predictable everywhere

Loading

Skeleton rows for lists, spinners for fetches, animated checklists for multi-step verification.

Empty

Every list has a real empty state with an icon, one line of guidance, and a primary action.

Error & recovery

Failed payment, wrong UPI, no SIM, blocked device, timeout — each with a calm explanation + retry path.

Trust signals

Verified-merchant badges, "no money was deducted", "we never ask for your PIN", RBI/UIDAI/DPDP cues.

Fraud prevention

First-time-payee caution, large-amount warning, daily-limit guard, payee name-as-per-bank confirm.

Feedback

Haptic tap on keys, success chime + confetti, low buzz on failure (sound/vibration helpers).

07 · Accessibility audit

Inclusive by default

In place

  • ✓Hit targets ≥ 44px (keys, nav, icon buttons).
  • ✓Body text ≥ 13px; primary content 14–16px.
  • ✓aria-label on icon-only buttons; role="switch" + aria-checked on toggles.
  • ✓Status conveyed by icon + label + colour (never colour alone).
  • ✓10-language support scaffolded for localisation.

Before GA — to add in code

  • !Verify AA contrast on muted text over tinted fills.
  • !Wire real focus management & screen-reader order (VoiceOver / TalkBack).
  • !prefers-reduced-motion to damp confetti / scratch / beam.
  • !Dynamic Type / font-scaling pass at 130%.

09 · Responsive guidelines

Frame-true, device-safe

Layout model

Designed at a 393×852 logical canvas (iPhone). Flex/grid with gap throughout — content reflows, never relies on fixed offsets.

Safe areas

Sticky docks & sheets use env(safe-area-inset-bottom); home indicator + dynamic island respected.

Scroll regions

One scroll container per screen (.screen-scroll); app bars & docks pinned outside it.

iOS ↔ Android

Swap the frame chrome + status bar; nudge nav-bar height & back affordance. Tokens, type, motion stay identical.

10 · Developer handoff framework

From prototype to production

Keep

  • ✓Token layer & component classes → map 1:1 to RN / Compose / SwiftUI themes.
  • ✓Screen inventory & nav contract → route table.
  • ✓Copy, microcopy & states → spec for engineering.

Replace

  • !window.DATA mocks → real API / state layer.
  • !Simulated OTP/PIN/scan → NPCI UPI SDK, device camera, secure keystore.
  • !In-browser Babel → native build (RN or platform-native).
  • !Demo triggers (₹1 fail, ₹2 reversal) → real gateway responses.

11 · Final premium polish

To feel truly production-ready

① Real assets

Merchant logos, bank marks & offer art in place of monograms/placeholders.

② Dark mode

The token layer already supports it — ship a dark variant for OLED.

③ Reduced-motion

Honour OS setting; offer a calm-mode for confetti & beams.

④ Skeleton parity

Match every skeleton to its loaded layout exactly to kill layout shift.

⑤ Haptic nuance

Distinct patterns for success / warning / error per platform guidelines.

⑥ Number transitions

Animate balance & coin count-ups for a premium feel.

⑦ Empty-to-rich

Personalise dashboard density as the user's history grows.

⑧ Localisation QA

Test long strings (Tamil, German) for truncation & wrap.

⑨ Performance

Virtualise long lists; lazy-load module bundles natively.
